iptables

防火墙错误导致的原因

  1. 开通httpd后，当前机器下使用curl可访问，但外部机器访问ip+端口则无法访问 curl报如下错误：curl: (7) Failed connect to 172.16.225.43:7001; No route to host

解决办法
—-> 在被访问机器下开放8888端口
—-> iptables -l INPUT -p tcp –dport 8888 -j ACCEPTt 8888 -j ACCEPT
